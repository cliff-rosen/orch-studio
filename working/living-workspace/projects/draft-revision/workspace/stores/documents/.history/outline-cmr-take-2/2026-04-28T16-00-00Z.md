# Outline — cmr take 2

Structural outline of `source-cmr-take-2.md`.

## §1 Goal

- Build an AI-assisted photo editing system for novices
- Two outcomes: stronger images + skill development
- Adapts along four axes (image, skill, outcome, edit stage)
- Reliable delivery requires orchestration layer
- Headline: LLM = communication; orchestration = control

## §2 Target User Experience

- User opens an image
- System gathers context
- Closed-loop interaction
- Loop continues until acceptable result
- Frame: this is not prompt-and-response

## §3 Why a Mega-Prompt Falls Short

- Pro photographer's process is silent and deep
- Mega-prompt approach: plausible but not good
- Two failure modes — orchestration addresses both differently

### Decomposition

- Mega-prompt asks LLM to do staged work in one pass
- Model satisfices: produces fluent output that resembles staged work
- Fix: orchestrator externalizes phase machine
- LLM invoked once per phase with scoped prompt
- Principle: never ask the LLM to do more than it has to do
